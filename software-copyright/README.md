# Software Copyright Skill (软著资料生成助手)

## 简介

专为**中国计算机软件著作权申请**设计的智能生成 Skill。采用**"规划 → 执行"**双阶段模式，深度分析项目源码，自动生成全套申请资料。

## 核心亮点

- **双阶段模式**：先生成《软著规划书》经用户确认，再精确生成文档
- **三件套一键生成**：软著主文档 + 代码鉴别材料（自动清洗/分页）+ 使用说明书（UI 级描述）
- **深度功能挖掘**：自动发现后台管理、运维监控等"隐形功能"
- **智能格式转换**：自动调用 Pandoc 转换 Word 格式
- **断点恢复**：长流程中断后可从上次完成处继续
- **广泛适配**：支持前端/后端/小程序/Monorepo 等多种项目类型

## 使用流程

```
第一步：信息收集 → 第二步：项目分析 → 第三步：读取文档
    ↓
第四步：生成规划书 → 【用户确认】
    ↓
第五步：生成主文档 → 第六步：生成代码文档 → 第七步：生成使用说明书
    ↓
第八步：输出与转换
```

> 详细执行规则请参阅 [SKILL.md](./SKILL.md)。

## 输出文件

| 文件 | 说明 | 后续操作 |
|------|------|----------|
| `{名称}_{日期}.md` | 软著主文档 | 填写申请表 |
| `{名称}_代码文档.docx` | 代码鉴别材料 | **调整字号为宋体小六/8pt**，设行号 |
| `{名称}_代码文档.md` | 代码鉴别材料源文件 | 备用 |
| `{名称}_使用说明书.md` | 使用说明书 | **补充截图** + 导出 PDF |

## 后续手动操作

1. **代码排版**：打开 `.docx` → 全选 → 宋体 8pt → 布局 → 行号 → 连续 → 页眉添加软件名和页码
2. **补充截图**：搜索 `【需补充截图】` 标记，粘贴实际运行截图
3. **打印盖章**：所有材料打印后加盖公章

## 常见问题

**Q: 代码不够 3000 行？**
A: Skill 自动检测并使用"全量模式"输出所有代码。

**Q: 没有 Pandoc？**
A: 只会生成 `.md` 格式，可用 Typora 或 VS Code Markdown PDF 插件手动导出。

**Q: 说明书里没有截图？**
A: AI 无法运行界面截图，文档中已预留 `【需补充截图】` 位置，请手动补充。
